화면 진입과 접근 제어

모든 라우트는 src/routes/Router.tsx에 선언합니다. 새 화면은 URL과 함께 어떤 레이아웃, 로딩 경계, 인증·역할 가드를 거치는지 결정해야 합니다.

flowchart TD

  Auth --> Login[login]
  Auth --> Signup[signup]
  Auth --> Find[find]

  Root --> NotFound["/* · NotFoundPage"]

가드와 레이아웃

경계적용 대상동작
MainRoute활동, 공지, 회원, 마이페이지MainLayout과 공통 콘텐츠 여백을 제공합니다.
AuthRoute/auth/*로그인 사용자를 /로 이동시키고 인증 화면 레이아웃을 제공합니다.
MyPageRoute/mypageaccess token이 없으면 /로 이동하고, 있으면 Suspense 경계를 제공합니다.
AdminRoute/admin/*ROLE_ADMIN 이상만 관리 사이드바와 화면에 접근합니다.
AdminNoticeRoute공지 작성·수정ROLE_ADMIN 미만이면 /notice로 이동합니다.
AdminBoardRoute활동 게시판 수정ROLE_TEAM_LEADER 미만이면 이전 화면으로 이동합니다.
AdminActivityPostRoute활동 게시글 수정ROLE_MEMBER 미만이면 이전 화면으로 이동합니다.

역할과 리다이렉트 기준

역할 순서적용 방식리다이렉트 예시
ROLE_WEB_MASTER → ROLE_ADMIN → ROLE_TEAM_LEADER → ROLE_MEMBERisRoleAboveOrEqual은 대상 역할과 같거나 상위 역할을 허용합니다.관리자·공지 가드는 지정 경로로 이동하고, 활동 수정 가드는 이전 화면으로 이동합니다.
인증되지 않은 사용자MyPageRoute는 access token을 확인합니다./mypage 접근 시 /로 이동합니다.
알 수 없는 경로최상위 와일드카드 라우트가 처리합니다.NotFoundPage를 렌더링합니다.

새 화면 추가 체크

  1. src/pages에 화면을 두고 src/pages/index.ts에서 노출합니다.
  2. Router.tsx에 URL을 추가하고 상위 레이아웃과 가드를 선택합니다.
  3. 로그인 또는 역할이 필요하면 기존 가드가 조건과 리다이렉트 방식에 맞는지 확인합니다.
  4. 데이터 조회나 변경이 있으면 데이터·상태의 Query·상태 기준을 적용합니다.
