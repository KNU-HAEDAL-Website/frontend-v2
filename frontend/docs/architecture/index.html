시작 구조 화면·접근 데이터·상태

기능을 안전하게 추가하기 위한 구조 안내

새 기능을 어느 계층에 두고, 어떤 라우트·권한·데이터 흐름을 함께 확인해야 하는지 안내합니다.

구조

페이지, 공통 컴포넌트, 서비스, 상태 계층의 책임과 의존 방향을 확인합니다.

화면·접근

새 화면의 URL, 레이아웃, 로그인·역할 조건을 결정합니다.

데이터·상태

API 호출, 캐시 무효화, 클라이언트 상태와 인증 흐름을 추가합니다.

갱신 기준

  • 계층 책임이나 디렉터리 경계가 바뀌면 구조를 갱신합니다.
  • 화면·라우트·레이아웃·가드·역할 조건이 바뀌면 화면·접근을 갱신합니다.
  • API·React Query·Zustand·인증 요청 흐름이 바뀌면 데이터·상태를 갱신합니다.
검증: 다이어그램 갱신이나 구조 검증이 필요하면 $architecture-diagram을 요청합니다. 스킬은 코드와 문서를 대조해 필요한 파일만 수정합니다.

릴리스마다 문서 최신성을 점검합니다. 이 문서는 저장소에서 직접 브라우저로 열람하는 용도입니다.
